## 1. 项目结构速览

```
vision-hub/
├── backend/          # Go 后端服务（Fiber + GORM + PostgreSQL + Redis）
├── app/              # Android 监护端（Kotlin + Compose）
├── hardware/         # 硬件固件（ESP32 + K210）
│   ├── esp32/esp32sense.ino    # ★ 最新 ESP32 固件
│   └── k210/main.py            # K210 代码
├── docs/             # 项目文档
└── apk/              # 构建好的 APK
```

**三端通信架构**：

```
硬件(ESP32+K210) ──WiFi──→ 后端(Go) ←──HTTP── Android APP
       ↑ 离线本地推理            ↑ 云端中转          ↑ 监护端
```

## 3. 后端部署（云服务器）

### 3.1 登录服务器

```bash
ssh root@47.94.146.53
# 密码见 _credentials.md → 云服务器
```

### 3.2 安装 Docker（如果还没装）

```bash
curl -fsSL https://get.docker.com | bash
systemctl enable docker
docker --version
```

### 3.3 拉取项目

```bash
# 在服务器上
cd /opt/visionguard
git clone https://gitee.com/taylorchengitee/vision-guard.git repo
```

部署文件在 `repo/cloud-deploy/` 目录下。

### 3.4 配置环境变量

在服务器上创建 `.env`：

```bash
# 在服务器上
cd /opt/visionguard/repo/cloud-deploy
# 从旧容器提取环境变量（如果之前部署过）
docker inspect visionguard-backend-1 --format '{{range .Config.Env}}{{println .}}{{end}}' | grep -vE '^(PATH=|TZ=)' > .env
# 或从模板创建
cp .env.example .env
nano .env
```

**`.env` 需要修改的地方**：

```ini
# Server
SERVER_PORT=3000

# PostgreSQL（Docker 部署用服务名）
DB_HOST=postgres          # ← 改！本地是 localhost，Docker 用服务名
DB_PORT=5432
DB_USER=visionhub
DB_PASSWORD=ff2aa51e15cc7e8e9e75c981f7af9e01   # ← 改！设复杂密码
DB_NAME=visionhub

# Redis（Docker 部署用服务名）
REDIS_HOST=redis           # ← 改！本地是 localhost，Docker 用服务名
REDIS_PORT=6379

# JWT
JWT_SECRET=你的随机64位密钥   # ← 改！用 openssl rand -hex 32 生成

# OCR Service（暂未对接）
OCR_SERVICE_URL=http://localhost:8001/ocr

# LLM（暂未对接，占位）
LLM_API_URL=http://localhost:8002/v1/chat/completions
LLM_API_KEY=sk-change-me

# Device Auth
DEVICE_UNIQUE_CODE=DEVICE_2026_ESP32_K210
DEVICE_XOR_KEY=4B
```

**关键区别：本地 vs 服务器**

| 配置项 | 本地开发 | 云服务器（Docker） |
|--------|---------|-------------------|
| DB_HOST | `localhost` | `postgres`（docker-compose 服务名） |
| REDIS_HOST | `localhost` | `redis`（docker-compose 服务名） |

### 3.5 启动服务

```bash
cd /opt/visionguard/repo/cloud-deploy

# 生产环境启动（含 PostgreSQL + Redis + 后端）
docker compose -f docker-compose.prod.yml up -d --build

# 查看日志
docker compose -f docker-compose.prod.yml logs -f backend

# 查看状态
docker compose -f docker-compose.prod.yml ps
```

### 3.6 验证后端运行

```bash
# 健康检查
curl http://47.94.146.53:3000/api/v1/healthz

# 应返回
# {"code":0,"message":"ok","data":{"status":"healthy"}}
```

### 3.7 常用运维命令

```bash
# 重启后端
docker compose -f docker-compose.prod.yml restart backend

# 查看后端日志
docker compose -f docker-compose.prod.yml logs -f backend

# 进入数据库
docker compose -f docker-compose.prod.yml exec postgres psql -U visionhub -d visionhub

# 进入 Redis
docker compose -f docker-compose.prod.yml exec redis redis-cli
```

## 4. 硬件配置修改

### 4.1 ESP32 固件要改的地方

文件：`hardware/esp32/esp32sense.ino`

```cpp
// ★ 第 35 行 — 服务器地址
const char* BASE_URL = "http://10.26.43.176:3000";
// 部署后改为：
const char* BASE_URL = "http://47.94.146.53:3000";

// ★ WiFi 热点（按实际环境修改）
const char* WIFI_SSID = "wuiPhone 16";
const char* WIFI_PASS = "12345ssDLH";
```

**WiFi 要求**（ESP32 硬件限制）：

| 要求 | 说明 |
|------|------|
| SSID | 必须英文（不支持中文编码） |
| 安全协议 | WPA2-Personal（不支持 WPA3） |
| 频段 | 2.4GHz（不支持 5GHz） |

### 4.2 烧录 ESP32

1. 用 Arduino IDE 打开 `hardware/esp32/esp32sense.ino`
2. 选择开发板：ESP32 Dev Module
3. 修改上面的 `BASE_URL` 和 WiFi 配置
4. 编译上传

### 4.3 K210 无需修改

`hardware/k210/main.py` 通过串口与 ESP32 通信，不直接联网，无需改。

## 7. 常见问题

### Q: 后端启动后连不上？
```bash
# 先检查容器状态
docker compose -f docker-compose.prod.yml ps
# 看后端日志
docker compose -f docker-compose.prod.yml logs backend
```

### Q: 数据库连接失败？
检查 `.env` 中 `DB_HOST=postgres`（不是 `localhost`），Docker 内部用服务名通信。

### Q: ESP32 连不上服务器？
1. 确认 WiFi 已连接（看串口日志）
2. 确认服务器 IP 可达：`ping 47.94.146.53`
3. 确认服务器 3000 端口开放

### Q: Android APP 连不上服务器？
1. 检查 `RetrofitClient.kt` 中 `BASE_URL` 是否改为服务器地址
2. 确认手机有网络
3. 确认服务器防火墙开放 3000 端口

### Q: 想回到本地开发模式？
反向操作三个配置：
- 后端 `.env`：`DB_HOST=localhost`，`REDIS_HOST=localhost`
- ESP32：`BASE_URL = "http://47.94.146.53:3000"`
- Android：`BASE_URL = "http://127.0.0.1:3000/"` + `adb reverse tcp:3000 tcp:3000`
