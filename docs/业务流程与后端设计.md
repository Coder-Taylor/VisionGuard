# 业务流程与后端设计

> 给学长的交付文档。覆盖全部接口、业务流程、数据库、认证体系、安全设计。

## 1. 实现概览

基于 `业务设计(1).md` 实现 **74 个 HTTP 路由 + 1 个健康检查**，覆盖 10 大业务模块，16 张数据库表。

| 模块 | 路由数 | 认证方式 | 核心功能 |
|---|:---:|---|---|
| 一、认证服务 | 8 | 无 | 用户注册登录、设备 challenge-response (XOR 0x4B)、JWT 签发/刷新 |
| 二、老人档案与监护关系 | 15 | UserAuth | 档案 CRUD、监护人邀请/接受/转让/移除、仪表盘 |
| 三、设备接入与安全注册 | 8 | 无+DeviceAuth | 设备激活、认证、信息记录、固件检测、数据上报 |
| 四、心跳与在线状态管理 | 5 | DeviceAuth+UserAuth | 心跳上报、在线状态、批量查询、离线检测（10s 扫描） |
| 五、设备绑定与解绑 | 7 | UserAuth | 设备搜索、发起/确认绑定、唯一约束校验、解绑/换绑 |
| 六、设备数据接收与存储 | 2 | 混合 | 健康数据接收（心率/血压/步数/血氧）、历史查询 |
| 七、告警事件管理 | 8 | 混合 | 7 种告警类型、去重窗口、状态管理（confirm/resolve/close）、统计 |
| 八、定位与设备状态展示 | 7 | UserAuth | 最新位置（Redis→DB）、轨迹、电子围栏、告警地图标记 |
| 九、药品识别与智能建议 | 7 | UserAuth | 图片上传、OCR 异步识别、LLM 建议生成、用户反馈 |
| 十、消息推送与通知 | 8 | UserAuth | 消息列表、已读标记、推送目标选择、优先级配置 (P0-P3) |

**技术栈**：Go 1.23 + Fiber v2 + GORM + PostgreSQL 16 + Redis 7

## 2. 接口详细清单

### 2.1 认证服务（模块一，8 路由）

| 方法 | 路径 | 请求 | 响应 | 说明 |
|------|------|------|------|------|
| POST | `/api/v1/device/challenge` | `{"deviceId"}` | `{challengeId, nonce, timestamp}` | XOR(device_code+nonce+timestamp, 0x4B)，Redis TTL 5min |
| POST | `/api/v1/device/verify` | `{deviceId, challengeId, sigin}` | `{jwt}` | 比对加密签名，签发设备 JWT (24h) |
| POST | `/api/v1/device/register` | `{deviceId, deviceModel, firmwareVersion, ip}` | DB 更新 | 设备首次接入注册，记录 IP |
| POST | `/api/v1/device/info` | `{deviceId, deviceModel, fwVersion}` | DB 更新 | 记录设备型号和固件版本 |
| POST | `/api/v1/device/log` | `{deviceId, logType, message}` | DB 写入 | 设备认证事件日志 |
| POST | `/api/v1/auth/register` | `{username, password, email, phone}` | 成功/失败 | bcrypt 哈希，8 字符最小密码，重复检查 |
| POST | `/api/v1/auth/login` | `{username, password}` | `{access_token, refresh_token, expires_in}` | JWT (1h) + refresh (30d)，8 次失败锁定 |
| POST | `/api/v1/auth/refresh` | `{refresh_token}` | `{access_token, refresh_token}` | 先创建新 token 再删除旧 token（翻转安全） |
| POST | `/api/v1/auth/logout` | `{refresh_token}` | 成功 | 删除 refresh_token |

**XOR 0x4B 加密实现**：
```go
// 明文 = device_code（32位随机字符串）+ nonce（10位随机）+ timestamp
// 逐字节 XOR 0x4B → hex 编码 → 与服务端比对
func xorEncrypt(input string) string {
    key := byte(0x4B)
    result := make([]byte, len(input))
    for i := 0; i < len(input); i++ {
        result[i] = input[i] ^ key
    }
    return hex.EncodeToString(result)
}
```

### 2.5 设备绑定与解绑（模块五，7 路由，UserAuth）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/device/:deviceId/search` | 搜索设备，返回 canBind 判断（status=registered + bindStatus=unbound + 非 disabled） |

## 3. 核心业务流程

### 3.1 设备开机 → 注册 → 绑定 → 正常工作

```
1. 设备出厂预烧录 serialNo → APP 扫描录入
2. POST /api/v1/device/activate → 云端发放 deviceID + deviceSecret
3. 设备上电 → POST /api/v1/device/register → 记录 IP
4. POST /api/v1/device/challenge → 获取 nonce+timestamp
5. 设备端 XOR(device_code + nonce + timestamp, 0x4B) → POST /api/v1/device/verify
6. 云端校验签名 → 签发设备 JWT (24h)
7. 监护人 APP 登录 → GET /api/v1/device/:id/search → 确认可绑定
8. APP POST /api/v1/binding/initiate → 创建绑定请求 (5min 超时)
9. 设备端 POST /api/v1/binding/confirm → 确认绑定 → 设备 bind_status=bound
10. 设备进入正常工作模式 → POST /api/v1/device/heartbeat (每 30s)
```

### 3.2 摔倒告警全链路

```
1. MPU6050 检测到摔倒 → ESP32 本地蜂鸣器紧急告警（离线可用）
2. 联网状态下 POST /api/v1/alert {deviceId, alertType:"fall", alertLevel:"critical", ...}
3. 云端：
   a. 去重检查（同一设备+同一类型+120s 窗口内）
   b. 查 bindings 表找 elderID
   c. 写入 alerts 表 + 查 guardianships 找所有监护人
   d. 为每个监护人创建 notification (P0, app+sms+voice_call)
4. APP 轮询 GET /api/v1/notifications → 弹窗 + 铃声
5. APP GET /api/v1/alert/:alertId → 查看详情（传感器数据、时间线）
6. 监护人 PUT /api/v1/alert/:alertId/status confirm → 标记已确认
```

### 3.3 药品识别全链路

```
1. 老人按按键 A → K210 拍照 → ESP32 上传
2. POST /api/v1/ocr/image {imageCategory:"medicine", fileUrl: OSS URL}
3. POST /api/v1/ocr/recognize {imageId, language:"zh"}
4. 云端异步处理（mock 3s）：
   a. 文字识别 → OCR 文本（阿司匹林肠溶片...）
   b. 药品匹配 → 通用名/规格/成分/适应症/用法/禁忌/匹配置信度
5. APP POST /api/v1/ocr/suggestion → LLM 建议生成（mock 5s）
6. APP 轮询 GET /api/v1/ocr/poll/:taskId → 进度追踪
7. OCR 结果 → 设备扬声器播报（stub）
```

### 3.4 设备离线检测与重连

```
1. 设备每 30s POST /api/v1/device/heartbeat
2. 后台 goroutine 每 10s 扫描 devices 表
3. last_heartbeat 超过 90s → 标记 offline → 记录 last_offline_at
4. 同时：auth_log (log_type="offline") + notification → 所有监护人
5. 设备恢复心跳 → 标记 online → 记录 last_online_at + auth_log (log_type="reconnect")
```

### 3.5 主监护人转让

```
1. 主监护人 POST /api/v1/elder/:elderId/primary/transfer {toUserId} → 创建转让记录
2. 被转让人 POST /api/v1/elder/:elderId/primary/confirm {transferId, accept:true}
3. 事务处理：
   - guardianships: fromUser 角色→normal, toUser 角色→primary
   - elders.created_by → toUser
   - transfers.status → completed
```

## 4. 认证体系

### 4.1 设备认证（XOR 0x4B Challenge-Response）

```
流程图：
1. 设备 → POST /api/v1/device/challenge {deviceId}
2. 服务端 → Redis 存储 XOR(device_code + nonce + timestamp, 0x4B), TTL 5min
3. 服务端 → 返回 {challengeId, nonce, timestamp}
4. 设备端 → 本地计算 XOR(device_code + nonce + timestamp, 0x4B)
5. 设备 → POST /api/v1/device/verify {deviceId, challengeId, sigin}
6. 服务端 → 比对 Redis 中的加密值 → 签发设备 JWT (24h)
```

设备 JWT claims：
```json
{"deviceId": "DEV_xxx", "iat": 1712345678, "exp": 1712432078, "type": "device"}
```

### 4.2 用户认证

- 注册：密码 ≥8 字符，bcrypt DefaultCost 哈希
- 登录：JWT HS256，1h 有效期，type="user"，含 jti 唯一标识
- 刷新：refresh_token 30d 有效期，bcrypt hash 存储，翻转机制（先创建新→再删除旧）
- 安全策略：同一用户连续 8 次登录失败（30min 窗口）→ 账户锁定 (status=locked)

用户 JWT claims：
```json
{"userId": 1, "username": "test_user", "jti": "abc123...", "iat": 1712345678, "exp": 1712349278, "type": "user"}
```

### 4.3 中间件

- **UserAuth**：提取 Bearer token → VerifyUserJWT → c.Locals("userId", uint)
- **DeviceAuth**：提取 Bearer token → VerifyDeviceJWT → c.Locals("deviceId", string)

## 5. 数据库设计

GORM AutoMigrate 启动时自动建 16 张表。所有字段有 `json` tag（camelCase）。敏感字段标记 `json:"-"` 不返回。

| 表名 | 用途 | 关键字段 |

## 7. 安全设计

| 措施 | 位置 | 说明 |
|------|------|------|
| XOR 0x4B Challenge-Response | `service/auth.go` | 逐字符 XOR 0x4B → hex 编码，Redis TTL 5min |
| bcrypt 密码哈希 | `service/auth.go` | DefaultCost，8 字符最小长度 |
| JWT HS256 | `service/auth.go` | 设备 24h / 用户 1h，type 字段区分 |
| Refresh Token 翻转 | `service/auth.go` | 先创建新 token 再删除旧（防崩溃丢失） |
| 登录失败锁定 | `service/auth.go` | 8 次/30min → Redis 计数 → status=locked |
| 心跳 deviceId 从 JWT 取 | `handler/device.go` | 不信任请求体中的 deviceId |
| 设备注册 IP 防伪造 | `handler/auth.go` | DeviceInfoRegister 用 `c.IP()` |
| 敏感字段不返回 | `model/*.go` | PasswordHash, DeviceCode, TokenHash → `json:"-"` |
| Challenge Redis TTL | `service/auth.go` | 5 分钟自动过期，单次使用 |
| 通知归属校验 | `handler/notification.go` | MarkRead 校验 `user_id` 匹配 |
| 邀请身份验证 | `service/elder.go` | AcceptInvitation 校验手机/邮箱匹配 |
| 绑定监护人权限 | `service/binding.go` | InitiateBinding 关联 eldest+user 的 guardianship |
| 类型断言安全 | 全部 handler | comma-ok 模式，失败返回 401 |
| 去重防并发 | `service/binding.go` | 绑定前检查 pending 状态，防重复请求 |
| JSON 响应统一格式 | 全部 handler | `code` + `message` + `data` 三层结构，camelCase |
