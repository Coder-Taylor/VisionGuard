## 一、设备完整接入流程（4 步）

### 流程概览

```
激活 → 首次注册 → Challenge-Response → 获取 JWT → 后续业务
(一次)   (一次)      (每次上线/过期)     (24h有效)
```

### 1.1 激活（设备首次使用，只需一次）

设备出厂时带有 `serialNo`（序列号）。激活后云端分配 `deviceId` 和 `deviceSecret`。**`deviceSecret` 必须保存在设备本地，后续认证要用。**

```
POST /api/v1/device/activate
```

**请求体**：
```json
{
  "serialNo": "SN_TEST_001",
  "model": "ESP32_K210",
  "mac": "AA:BB:CC:DD:EE:FF",
  "hwVersion": "1.0",
  "fwVersion": "1.0.0",
  "timestamp": 1714567890,
  "sign": "test_sign"
}
```

| 字段 | 说明 |
|------|------|
| serialNo | 设备序列号，硬件预置 |
| model | 设备型号 |
| mac | MAC 地址 |
| hwVersion | 硬件版本 |
| fwVersion | 固件版本 |
| timestamp | 当前时间戳（秒） |
| sign | 保留字段 |

**返回**：
```json
{
  "code": 0,
  "message": "device registered",
  "data": {
    "deviceId": "DEV_a1b2c3d4",
    "deviceSecret": "8f3a2d1c...（32位随机字符串）",
    "expiresAt": "2027-05-04T23:59:59Z"
  }
}
```

⚠️ **ESP32 必须将 `deviceId` 和 `deviceSecret` 存入 NVS/EEPROM**，后续认证需要。

### 1.2 首次接入注册（设备每次上电）

告知云端设备已上线，记录 IP 和基础信息。

| 字段 | 说明 |
|------|------|

**返回**：

### 1.3 Challenge-Response 认证（获取 JWT，每次上线/过期时执行）

这是最关键的步骤。认证后获取 JWT，之后所有受保护接口都需要带 JWT。

#### Step 1：请求挑战

```
POST /api/v1/device/challenge
```

**请求体**：
```json
{
  "deviceId": "DEV_a1b2c3d4"
}
```

**返回**：
```json
{
  "challengeId": "c_x9k2m5",
  "nonce": "a8KpQ21xYz",
  "timestamp": 1714567890
}
```

| 字段 | 说明 |
|------|------|
| challengeId | 本次挑战的 ID |
| nonce | 10 位随机字符串 |
| timestamp | 服务端时间戳（秒） |

#### Step 2：计算 XOR 签名

```cpp
// ESP32 代码
String calcSign(String deviceSecret, String nonce, long timestamp) {
  String plaintext = deviceSecret + nonce + String(timestamp);
  String result = "";
  for (int i = 0; i < plaintext.length(); i++) {
    char c = plaintext[i] ^ 0x4B;
    // 转成 hex 字符串（2字符表示1字节）
    char hex[3];
    sprintf(hex, "%02x", (unsigned char)c);
    result += hex;
  }
  return result;
}
```

**计算公式**：`HEX(每个字符 XOR 0x4B)`，输入是 `deviceSecret + nonce + timestamp`

示例：
```
deviceSecret = "abc123..."
nonce = "a8KpQ21xYz"
timestamp = 1714567890
plaintext = "abc123...a8KpQ21xYz1714567890"
sign = 对 plaintext 逐字节 XOR 0x4B，然后 hex 编码
```

#### Step 3：验证签名，获取 JWT

```
POST /api/v1/device/verify
```

**请求体**：
```json
{
  "deviceId": "DEV_a1b2c3d4",
  "challengeId": "c_x9k2m5",
  "sigin": "af3c8d...（Step 2 计算出的 hex 字符串）"
}
```

**返回**：
```json
{
  "jwt": "eyJhbGciOiJIUzI1NiIs..."
}
```

✅ **JWT 有效期 24 小时**。ESP32 应保存 JWT，后续请求在 Header 中携带：

```
Authorization: Bearer eyJhbGciOiJIUzI1NiIs...
```

JWT 过期后（返回 401），重新执行 1.3 的 challenge-response 流程获取新 JWT。

## 二、日常业务接口（需 JWT）

### 2.1 心跳上报（每 30 秒）

保持在线状态 + 同步电量/信号/位置。

```
POST /api/v1/device/heartbeat
Authorization: Bearer <jwt>
```

**请求体**：
```json
{
  "deviceId": "DEV_a1b2c3d4",
  "timestamp": 1714567890,
  "battery": 85,
  "rssi": -55,
  "location": {
    "lat": 31.2304,
    "lng": 121.4737
  }
}
```

| 字段 | 说明 |
|------|------|
| timestamp | 设备当前时间戳（秒） |
| battery | 电量百分比（0-100） |
| rssi | WiFi 信号强度（dBm） |
| location.lat | 纬度 |
| location.lng | 经度 |

⚠️ **deviceId 必须填写，但服务端以 JWT 中的为准**（不信任 body 中的 deviceId）。

**返回**：
```json
{
  "code": 0,
  "message": "heartbeat ack",
  "data": {
    "serverTime": 1714567890,
    "nextInterval": 30
  }
}
```

**离线判定**：超过 90 秒未收到心跳 → 云端标记设备离线 → 通知监护人。

## 四、OCR 相关接口（药品识别）

> **两种上传方式**：硬件通过 multipart/form-data 直传 JPEG 二进制（deviceAuth），APP 通过 JSON 传 base64（userAuth）。

### 4.1 硬件 JPEG 直传上传（替代旧版 /upload_image）

```
POST /api/v1/device/ocr/image
Authorization: Bearer <deviceJWT>
Content-Type: multipart/form-data
```

| 表单字段 | 类型 | 说明 |
|----------|------|------|
| `image` | file | JPEG 二进制文件 |
| `deviceId` | string | 设备 ID（可选，JWT 中取） |
| `elderId` | string | 老人 ID（可选） |
| `category` | string | 图片类别，默认 "medicine" |

**返回**：
```json
{
  "code": 0,
  "message": "image uploaded",
  "data": {
    "taskId": "TASK_abc123...",
    "status": "doubao_recognizing",
    "stage": "豆包 AI 正在识别药品..."
  }
}
```

上传后服务端异步调用豆包 API 识别，结果存入 DB，硬件通过轮询获取。

> **ESP32 实现注意事项**（2026-05-08 修复）：
> 
> **不要使用 `HTTPClient + getStreamPtr()` 流式上传模式！** 以下写法会在 ESP32 上触发 `LoadProhibited` 崩溃重启：
> ```cpp
> // ❌ 错误写法（会崩溃）
> http.begin(client, url);
> http.addHeader("Content-Length", String(totalLen));
> http.POST("");                        // 发送空 body，库立即尝试读响应
> WiFiClient* stream = http.getStreamPtr();  // 服务器未响应 → _tcp=NULL → 空指针!
> stream->print(...);                   // 💥 LoadProhibited 崩溃
> ```
> 
> **原因**：`http.POST("")` 发送空 body 后，HTTPClient 库立即尝试读取服务器 HTTP 响应。但服务器看到 `Content-Length` 还在等待完整 body 数据，不会发送响应。库超时后内部 `_tcp` 指针被清为 NULL，后续写入触发空指针访问。
> 
> **正确写法**：用 `WiFiClient` 直连，malloc 拼装完整 multipart body 后一次性 `write()`：
> ```cpp
> // ✅ 正确写法（参考 esp32sense.ino 最新版）
> WiFiClient client;
> client.connect(host, 80);
> // 拼装完整 body 到 malloc buffer
> uint8_t* body = (uint8_t*)malloc(bodyLen);
> // ... memcpy 各部分到 body ...
> client.print("POST /vg/api/v1/device/ocr/image HTTP/1.1\r\n...");
> client.write(body, bodyLen);  // 一次性发送
> free(body);
> // 读取响应...
> client.stop();
> ```

## 六、ESP32 需要做的改动

### 6.1 必须改的

1. **认证流程重写**：旧版是静态 XOR 一次性发给 `/device_register`，新版是四步流程
2. **保存 deviceSecret**：激活返回的 32 位随机字符串必须存 NVS
3. **实现 XOR+Hex 签名计算**：见 1.3 节 Step 2 的 C++ 代码
4. **JWT 管理**：verify 成功后保存 JWT，后续请求在 Header 加 `Authorization: Bearer <jwt>`
5. **JWT 过期处理**：收到 401 响应 → 重新执行 challenge-response 获取新 JWT
6. **心跳**：新增每 30 秒的 heartbeat，携带电量/信号/位置
7. **路径变更**：所有接口路径改为 `/api/v1/...` 前缀

### 6.2 可以不改的

- **告警内容格式**：`/upload_alarm` → `/api/v1/alert`，字段基本兼容
- **绑定状态轮询**：逻辑不变，路径改为 `/api/v1/device/<deviceId>/binding`
- **ESP32 ↔ K210 串口协议**：完全不涉及，保持不变

### 8.3 查找本机局域网 IP

硬件 ESP32 需要通过 WiFi 连接你的电脑，需要知道电脑的 IP 地址。

**Windows（PowerShell）：**
```powershell
ipconfig | findstr "IPv4"
```

**macOS / Linux：**
```bash
ifconfig | grep "inet " | grep -v 127.0.0.1
```

找到 `无线局域网适配器 Wi-Fi` 或 `en0` 下的 IP，例如 `192.168.1.105`。

### 8.4 开放 Windows 防火墙

**以管理员身份打开 PowerShell**，执行：

```powershell
New-NetFirewallRule -DisplayName "VisionGuard 3000" -Direction Inbound -Protocol TCP -LocalPort 3000 -Action Allow
```

验证规则已添加：
```powershell
Get-NetFirewallRule -DisplayName "VisionGuard 3000" | Format-List Name, Enabled, Action
```

### 8.5 让硬件连接

1. 确保 ESP32 和你的电脑连接**同一个 WiFi 热点**
2. 在 ESP32 固件中将服务器地址设为 `http://<你的IP>:3000`
3. 硬件上电后按标准流程执行：

```
POST /api/v1/device/activate     → 激活，保存 deviceId + deviceSecret
POST /api/v1/device/register     → 首次接入注册
POST /api/v1/device/challenge    → 请求挑战码
POST /api/v1/device/verify       → XOR 签名验证，获取 JWT
POST /api/v1/device/heartbeat    → 开始心跳（每 30s）
POST /api/v1/alert               → 上报告警（摔倒/障碍等）
```

### 8.6 用 curl 模拟硬件测试

如果硬件还没就绪，可以用 curl 在另一台电脑（或同一台电脑另一个终端）模拟硬件行为：

```bash
# === 设置变量 ===
BASE="http://<你的IP>:3000"
# 生成唯一序列号（避免和已有设备冲突）
SN="SN_TEST_$(date +%s)"

# === Step 1: 激活设备 ===
echo "=== 1. 激活设备 ==="
RESP=$(curl -s -X POST $BASE/api/v1/device/activate \
  -H "Content-Type: application/json" \
  -d "{\"serialNo\":\"$SN\",\"model\":\"ESP32_K210\",\"mac\":\"AA:BB:CC:DD:EE:FF\",\"hwVersion\":\"1.0\",\"fwVersion\":\"1.0.0\",\"timestamp\":$(date +%s),\"sign\":\"test\"}")
echo $RESP

DEVICE_ID=$(echo $RESP | python3 -c "import sys,json; print(json.load(sys.stdin)['data']['deviceId'])")
DEVICE_SECRET=$(echo $RESP | python3 -c "import sys,json; print(json.load(sys.stdin)['data']['deviceSecret'])")
echo "deviceId=$DEVICE_ID"
echo "deviceSecret=$DEVICE_SECRET"

# === Step 2: 首次注册 ===
echo "=== 2. 首次注册 ==="
curl -s -X POST $BASE/api/v1/device/register \
  -H "Content-Type: application/json" \
  -d "{\"deviceId\":\"$DEVICE_ID\",\"deviceModel\":\"ESP32_K210\",\"firmwareVersion\":\"1.0.0\"}"

# === Step 3: 请求挑战 ===
echo -e "\n=== 3. 请求挑战 ==="
RESP=$(curl -s -X POST $BASE/api/v1/device/challenge \
  -H "Content-Type: application/json" \
  -d "{\"deviceId\":\"$DEVICE_ID\"}")
echo $RESP

CHALLENGE_ID=$(echo $RESP | python3 -c "import sys,json; print(json.load(sys.stdin)['challengeId'])")
NONCE=$(echo $RESP | python3 -c "import sys,json; print(json.load(sys.stdin)['nonce'])")
TIMESTAMP=$(echo $RESP | python3 -c "import sys,json; print(json.load(sys.stdin)['timestamp'])")

# === Step 4: XOR 签名并验证 ===
echo "=== 4. XOR 签名验证 ==="
# 明文 = deviceSecret + nonce + timestamp
PLAINTEXT="${DEVICE_SECRET}${NONCE}${TIMESTAMP}"

# Python XOR 计算（和硬件 C++ 代码等效）
SIGN=$(python3 -c "
import binascii
plaintext = '$PLAINTEXT'
key = ord('\x4b')
result = bytes([ord(c) ^ key for c in plaintext])
print(binascii.hexlify(result).decode())
")

RESP=$(curl -s -X POST $BASE/api/v1/device/verify \
  -H "Content-Type: application/json" \
  -d "{\"deviceId\":\"$DEVICE_ID\",\"challengeId\":\"$CHALLENGE_ID\",\"sigin\":\"$SIGN\"}")
echo $RESP

JWT=$(echo $RESP | python3 -c "import sys,json; print(json.load(sys.stdin)['jwt'])")
echo "JWT: ${JWT:0:30}..."

# === Step 5: 心跳 ===
echo -e "\n=== 5. 心跳上报 ==="
curl -s -X POST $BASE/api/v1/device/heartbeat \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $JWT" \
  -d "{\"deviceId\":\"$DEVICE_ID\",\"timestamp\":$(date +%s),\"battery\":85,\"rssi\":-55,\"location\":{\"lat\":31.2304,\"lng\":121.4737}}"

# === Step 6: 上报告警 ===
echo -e "\n=== 6. 上报告警 ==="
curl -s -X POST $BASE/api/v1/alert \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $DEVICE_JWT" \
  -d "{\"deviceId\":\"$DEVICE_ID\",\"timestamp\":$(date +%s),\"alertType\":\"fall\",\"alertLevel\":\"critical\",\"description\":\"检测到疑似摔倒事件\",\"location\":{\"lat\":31.2304,\"lng\":121.4737}}"

echo -e "\n\n=== 测试完成 ==="
```

### 8.7 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| `connect: Connection refused` | 后端未启动或端口不对 | 确认 `go run cmd/server/main.go` 已执行 |
| `connect: Connection timed out` | 防火墙阻挡 | 执行 8.4 节的防火墙命令 |
| `connect: Network is unreachable` | 不在同一网络 | 确认 ESP32 和电脑连同一个 WiFi，电脑 IP 正确 |
| `challenge expired` | 超过 5 分钟才验证 | 重新请求 challenge |
| `invalid signature` | XOR 计算错误 | 检查 deviceSecret 是否正确，明文拼接顺序对不对 |
| 401 Unauthorized | JWT 过期（设备 24h） | 重新执行 challenge → verify 获取新 JWT |
