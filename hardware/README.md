## 项目概述
本项目基于 **ESP32 + K210** 实现智能助行设备，集成**视觉识别、障碍物方位检测、摔倒报警、GPS 定位、环境录音、云端返回文本+本地语音播报**等功能，适用于助行、安防、健康监测、户外辅助等场景。

**核心特点：**
- K210 负责视觉采集、AI 推理、障碍物方位识别、拍照与图片传输
- ESP32 负责 WiFi 联网、云端通信、多传感器数据采集、本地语音合成
- **云端只返回文本，不传输音频**，设备端使用本地千字 WAV 字库播报
- 响应更快、流量更省、离线提示音可用、系统更稳定可靠

---

# 硬件组成
- **ESP32**：主控芯片、WiFi 通信、串口管理、传感器驱动、音频播放
- **K210**：视觉采集、AI 模型运行、障碍物检测、拍照、JPEG 压缩传输
- **MPU6050**：陀螺仪/加速度计，实现摔倒检测与姿态判断
- **VL53L5CX**：多区域激光雷达，实现近距离障碍物测距与避障
- **GPS/北斗模块**：实时定位，所有报警与上传数据均带经纬度
- **I2S 音频模块 + 喇叭**：本地语音播报输出
- **SPIFFS 闪存**：存储本地汉字语音库与提示音 WAV 文件
- **按键 A**：手动触发拍照识别功能

# 通信架构
## 1. ESP32 ↔ K210 串口通信
- 波特率：**115200**
- 格式：ASCII 指令 + 二进制图片流

### ESP32 → K210 指令
| 指令 | 功能 |
|------|------|
| `$TAKE_PHOTO!` | 触发 K210 拍照、压缩并回传图片 |

### K210 → ESP32 指令
| 指令 | 功能 |
|------|------|
| `$LEFT!` | 检测到障碍物在左侧 |
| `$MID!` | 检测到障碍物在正前方 |
| `$RIGHT!` | 检测到障碍物在右侧 |
| `$IMG_START:长度!` | 开始传输图片，携带字节长度 |
| `$IMG_END!` | 图片传输完成 |

---

## 2. ESP32 ↔ 云端 HTTP 接口
**服务器基础地址：** `http://172.20.10.3:3000`

| 接口路径 | 请求方法 | 功能说明 |
|---------|----------|----------|
| `/device_register` | POST | 设备上电联网后注册 |
| `/check_bind` | GET | 查询设备是否完成平台绑定 |
| `/upload_image` | POST | 上传 K210 拍摄的图片 |
| `/get_ocr_text` | GET | 获取云端识别后的文本结果 |
| `/upload_alarm` | POST | 上传摔倒/障碍物报警信息 |
| `/upload_audio` | POST | 上传环境录音数据 |

# 功能说明
## 1. 拍照识别（按键 A 触发）
1. 按下按键 → ESP32 发送 `$TAKE_PHOTO!`
2. K210 拍摄 → JPEG 压缩 → 串口分块传输
3. ESP32 接收完整图片 → 上传云端
4. 云端识别完成 → 返回纯文本结果
5. ESP32 **本地逐字朗读 WAV 语音库**，无网络延迟

## 2. 实时障碍物检测与语音提示
- K210 实时 AI 检测障碍物方位
- ESP32 自动播放本地提示音：**左 / 中 / 右**
- 激光雷达检测到近距离危险 → 触发避障提示 + 云端报警

## 3. 摔倒检测与报警
- MPU6050 实时计算姿态角
- 角度异常 → 判断为摔倒
- 本地播放摔倒提示音 + **上传带 GPS 定位的报警信息**

## 4. GPS 北斗定位
所有上传云端的数据（注册、报警、录音、图片）均携带**实时经度、纬度**，支持平台定位与轨迹查看。

## 5. 环境声音检测与录音上传
- 检测环境声音强度
- 声音触发 → 自动开始录音
- 静音超时（1.2秒）→ 自动停止并上传音频

## 6. 本地文本转语音（核心优势）
- 语音库存放于 ESP32 SPIFFS：`/words/xx.wav`
- 云端返回任意中文文本 → 设备逐字朗读
- 不依赖云端音频流，**零延迟、低流量、高可靠**

# 设备运行流程
1. 设备上电初始化
2. 自动连接 WiFi → 向云端注册
3. 循环查询绑定状态 → 绑定成功进入工作模式
4. 正常运行：
   - 障碍物实时检测 + 语音提示
   - 摔倒检测 + 报警上传
   - 环境声音检测 + 录音上传
5. 按键 A 触发：拍照 → 上传 → 识别 → 本地语音播报

# 云端对接规范（必须实现）
## 1. `/device_register`
- 接收设备注册信息（含唯一码、加密 token）
- 完成设备合法性校验

## 2. `/check_bind`
- 返回纯文本
- 已绑定：返回包含 `bind_success`
- 未绑定：返回其他内容

## 3. `/upload_image`
- 接收 JPEG 二进制流
- 进行识别/分析

## 4. `/get_ocr_text`
- 返回**纯中文文本**
- 示例：`药品识别成功，请按时服药`

## 5. `/upload_alarm`
- 接收 JSON 格式报警信息
- 包含设备号、类型、角度/距离、经纬度

## 6. `/upload_audio`
- 接收音频二进制流
- 存储或转码

# 安全认证说明（云端必须实现）
## 设备唯一标识
```
DEVICE_2026_ESP32_K210
```

## 加密密钥
```
0x4B
```

## 加密算法
对唯一码字符串**逐字符异或 0x4B**，生成 token，云端必须校验。

---

# 注意事项
1. ESP32 必须先烧录 **SPIFFS 字库与提示音**，否则无声音
2. WiFi 名称与密码在代码中配置修改
3. K210 与 ESP32 串口接线必须稳定，避免图片传输失败
4. 云端接口路径、参数、格式必须与文档完全一致
5. 设备仅在**绑定成功 + WiFi 在线**时上传数据
6. 音频文件必须为固定格式 WAV，否则无法播放

# 云端服务端代码

本仓库已包含匹配硬件与监护端新架构的 FastAPI 服务端：

```text
src/server/main.py
requirements-server.txt
scripts/run_server.sh
docs/fastapi_device_server.md
```

服务端分两层：

```text
硬件兼容接口 V1：保留 /device_register、/check_bind、/upload_image、/get_ocr_text、/upload_alarm、/upload_audio
APP增强接口 V1.1：新增用户注册登录、老人信息、在线设备、绑定解绑、告警历史、心跳、定位上传
```

本地调试：

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements-server.txt
./scripts/run_server.sh
```

默认监听：

```text
http://0.0.0.0:3000
```

Linux 服务器正式部署时使用 PostgreSQL：

```bash
export DATABASE_URL='postgresql://blind_assist_user:change_me@127.0.0.1:5432/blind_assist'
export REQUIRE_POSTGRES=1
export AUTO_BIND_ON_REGISTER=0
uvicorn src.server.main:app --host 0.0.0.0 --port 8888
```

详细说明见 `docs/fastapi_device_server.md`。
