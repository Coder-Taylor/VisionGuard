# VisionGuard 云服务器部署说明（三端交付包）

## 文件说明

| 文件/目录 | 作用 |
|-----------|------|
| `docker-compose.prod.yml` | Docker Compose 生产环境配置（PostgreSQL + Redis + Backend） |
| `Dockerfile` | 后端 Go 服务多阶段构建 |
| `deploy.sh` | 一键部署脚本 |
| `.env.example` | 环境变量模板（复制为 `.env` 后修改） |
| `internal/` | 后端 Go 源码（72 路由 + 16 表） |
| `cmd/` | 入口 main.go |
| `android/` | ★ Android 监护端完整源码 + 预构建 APK |
| `hardware/` | ★ 硬件固件完整源码（ESP32 + K210） |

---

## 快速部署（3 步）

### 1. 上传到服务器

```bash
# 在本地打包上传
cd vision-hub
scp -r submission/* root@47.94.146.53:/opt/visionguard/

# 或者用 rsync（推荐）
rsync -avz --delete --exclude='.env' submission/ root@47.94.146.53:/opt/visionguard/
```

### 2. 配置环境变量

```bash
ssh root@47.94.146.53
cd /opt/visionguard

# 首次部署：复制并编辑 .env
cp .env.example .env

# 确认关键配置（通常无需修改）
#   DB_HOST=postgres      ← Docker 内部服务名，不要改！
#   REDIS_HOST=redis      ← Docker 内部服务名，不要改！
#   JWT_SECRET=...        ← 已预填
#   DB_PASSWORD=...       ← 已预填
```

### 3. 构建并启动

```bash
cd /opt/visionguard
chmod +x deploy.sh
./deploy.sh
# 或手动:
docker compose -f docker-compose.prod.yml down
docker compose -f docker-compose.prod.yml up -d --build
```

### 验证

## 交付内容

### Android APP

- **源码**：`android/` 目录（Kotlin + Compose 完整工程）
- **预构建 APK**：`android/apk/VisionGuard-v1.4.apk`（103MB）
- **已配置**：连接云服务器 `47.94.146.53:3000`
- **安装**：`adb install android/apk/VisionGuard-v1.4.apk` 或传文件到手机直接安装
- **自行构建**：见 `android/README.md`

### 硬件固件

- **ESP32**：`hardware/esp32/esp32sense.ino` + `wordmap.h`
  - 修改 WiFi 名（必须英文，2.4GHz）+ 密码 + serverUrl
  - Arduino IDE 上传到 ESP32
- **K210**：`hardware/k210/main.py` + `detect.kmodel`
  - MaixPy IDE 或 kflash 烧录到 K210
